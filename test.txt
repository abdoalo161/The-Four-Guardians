This is a highly detailed analysis of the Rock, Paper, Scissors Unity project, suitable for an AI assistant to understand and reconstruct the project.

## Project Overview

This is a multiplayer Rock, Paper, Scissors game built with Unity. It features a main menu, a lobby system for discovering and joining games on a local network, and the core game scene. The project utilizes Unity's Netcode for GameObjects for networking, the Universal Render Pipeline (URP) for graphics, and the new Input System for player controls.

## Technology Stack

*   **Game Engine:** Unity
*   **Primary Language:** C#
*   **Networking:** Unity Netcode for GameObjects
*   **Transport Layer:** Ruffles (a community-created, performance-focused transport for Netcode)
*   **Rendering:** Universal Render Pipeline (URP)
*   **Input:** Unity Input System
*   **UI:** Unity UI (UGUI) with TextMeshPro

## Scene Breakdown

*   **`MainMenuScene.unity`**: The entry point of the game. It contains the main menu UI with buttons for "Host Game," "Join Game," and "Options."
*   **`LobbyScene.unity`**: This scene is for finding and joining games. It displays a list of discovered games on the local network and allows for manual connection via IP address.
*   **`GameScene.unity`**: The main game scene where the Rock, Paper, Scissors match takes place. It includes the UI for displaying player choices, scores, and game results.
*   **`OptionsScene.unity`**: A scene for adjusting game settings, such as music and sound effect volume.

## Script-by-Script Analysis

### Core Gameplay & Networking

*   **`RPSGameManager.cs`**: The heart of the game logic. It's a `NetworkBehaviour` that manages the game state, player scores, and round progression. It's responsible for determining the winner of each round and the overall match. It communicates with the UI through static events.
*   **`RPSPlayer.cs`**: Represents a player in the game. It's a `NetworkBehaviour` that holds the player's ID and their current choice (Rock, Paper, or Scissors) in a `NetworkVariable`. It sends the player's choice to the server via a `ServerRpc`.
*   **`RPSChoice.cs`**: A simple `enum` that defines the possible choices: `None`, `Rock`, `Paper`, `Scissors`.
*   **`RPSNetworkDiscovery.cs`**: Implements network discovery using UDP broadcasting. It allows the game to find and join games hosted on the same local network without requiring the user to manually enter an IP address.

### UI & Scene Management

*   **`MainMenuManager.cs`**: Manages the main menu UI and scene transitions. It handles the button clicks for hosting, joining, and going to the options scene.
*   **`LobbyManager.cs`**: Manages the lobby UI. It displays the list of discovered games and provides a way to manually join a game with an IP address.
*   **`RPSUIManager.cs`**: Manages all the in-game UI elements. It updates the score display, shows player choices, and displays round and game-end results. It subscribes to events from `RPSGameManager` to stay in sync with the game state.
*   **`OptionsManager.cs`**: Manages the options menu. It allows the player to adjust music and sound effect volume and saves these settings using `PlayerPrefs`.

### Audio

*   **`RPSAudioManager.cs`**: A singleton that manages all audio in the game. It has `AudioSource` components for music and sound effects and provides methods to play various sounds. It subscribes to game events to play sounds at the appropriate times (e.g., when a choice is made, a round is won, or the game ends).

### Utilities

*   **`DebugConsole.cs`**: A simple, on-screen console for displaying debug logs. This is a helpful utility for development and debugging.
*   **`DontDestroy.cs`**: A simple script that calls `DontDestroyOnLoad` on the `GameObject` it's attached to. This is used to keep objects like the `RPSAudioManager` and `NetworkManager` persistent across scene changes.
*   **`UnityMainThreadDispatcher.cs`**: A utility for executing code on Unity's main thread from other threads. This is crucial for networking callbacks, which often run on separate threads, to safely interact with Unity's API (which is not thread-safe).

## Package Dependencies (`Packages/manifest.json`)

This project relies on a number of Unity packages:

*   **`com.unity.netcode.gameobjects`**: The core package for Unity's Netcode for GameObjects.
*   **`com.community.netcode.extensions`**: Community-provided extensions for Netcode.
